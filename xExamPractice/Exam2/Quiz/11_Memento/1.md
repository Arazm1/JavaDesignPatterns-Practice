1. What is the primary intent of the Memento design pattern?

# To capture and externalize and object's internal state without violating encapsulation, allowing the object to be restored to this state later.


2. In the Memento pattern, which component is responsible for safekeeping the memento objects but never examining their contents?
# Caretaker


3. In Java, what is a common and robust technique to ensure that the Caretaker cannot modify the state stored inside a Memento, preserving strict encapsulation?
# Implementing the Memento as a private or package-private inner class within the Originator class.


4. Which category of the Gang of Four (GoF) design patterns does the Memento pattern belong to?
# Behavioral


5. What is a significant performance or resource drawback to watch out for when using the Memento pattern heavily?
# It can consume significant amounts of memory if the Originators state is large and mementos are created frequently.


6. If you are implementing an undo/redo system in Java, how are the Command and Memento patterns typically used together?
# A Command object can store A Memento representing the state of the system before the Command was executed, allowing the Command to easily reverse its effects.


7. What is the primary responsibility of the Originator in the Memento pattern?
# To create a Memento containing a snapshot of its current internal state, and to use a Memento to restore its state.


8. In Java, standard object serialization (java.io.Serializable) is sometimes used as a quick way to implement the Memento pattern. What is a key disadvantage of this approach compared to a custom Memento?
# Slow, fragile to class structure changes, and might expose sensitive internal data if not carefully managed.


9. Which two interfaces are conceptually described by the Memento pattern to enforce encapsulation?
# "Wide" interface for the Originator (allowing full access to state).
# "Narrow" interface for the Caretaker (allowing no access to state).


10. Consider a Java text editor implementing Memento. If the user types "Hello", saves a Memento, types " World", and then hits undo. What happens exactly?
# Caretaker passes the saved Memento back to the Originator, which reads the Memento and overwrites its current text with "Hello".
